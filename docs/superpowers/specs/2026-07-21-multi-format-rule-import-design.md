# 多格式双数据源导入设计

## 目标

将战场频谱智能指配工作台调整为“用频需求数据 + 禁用保护/规则数据”双输入模式。页面不再展示算法名称或算法选择；系统使用唯一的频率指配计算入口，由“启动计算”按钮触发。

## 用户界面

网页端与 PySide6 桌面端保持相同的三栏布局与深蓝工程软件风格。

左栏保留少量计算配置：可用频谱范围、频率判定容差和同频/邻频/三阶互调规则开关。结果指标不作为输入参数显示。

左栏数据接入区提供两个独立按钮：

1. 导入用频需求数据：支持 `.xlsx`、`.xls`、`.csv`；导入后显示文件名和记录数。
2. 导入禁用保护/规则数据：支持 `.docx`、`.xlsx`、`.xls`、`.csv`、`.json`、`.txt`；导入后显示文件名、解析出的规则数和不可解析项目数。

“生成模拟数据”一次性生成可运行的用频需求和保护规则。只有两类数据都有效时，“启动计算”可用。启动后保留约三秒的加载进度效果，不显示算法名称。

中栏在导入用频需求后展示原始冲突组合；启动计算后切换为计算后的冲突组合。右栏提供结果 Excel 下载，并显示原始与计算后冲突数的对比。导出的结果工作簿保留原始格式，仅填写“建议”列。

## 数据模型与解析

新增统一的 `ProtectionRule` 内部模型，至少包含：频率表达式、归一化频段或频点、用途/类别、保护要求、来源文件和解析状态。所有频率在解析后统一换算为 MHz。

用频需求数据继续采用既有的 12 列业务模型。`.xlsx` 与 `.xls` 自动寻找包含“序号、型号名称、工作频率”的表头；`.csv` 使用同一列名校验。

保护规则数据按来源解析：

- `.docx`：读取正文和表格，提取频段、频点和 `±` 公差表达式，并保留相邻文字中的保护要求。
- `.xlsx`、`.xls`、`.csv`：识别“频率”必填列；可选列为“用途/类别”“保护要求”“保护距离/带宽”。
- `.json`：接受规则对象数组，每项至少含 `frequency`，其余字段可选。
- `.txt`：一行一条规则，格式为 `频率 | 用途/类别 | 保护要求`，后两段可省略。

频率表达式先支持连续频段（`2523-2654MHz`）、多频段/多频点（逗号分隔）、单位混用（GHz/MHz/kHz）和带公差频点（`13525±10kHz`）。无法解析的条目不阻止其他有效规则导入，但在界面明确提示。

## 计算边界

本次仍是展示阶段，唯一计算入口不向用户暴露算法选择。它使用导入的两类数据统一生成冲突前后结果和“建议”文本；真实算法后续替换该内部入口，无需改动界面或文件接口。

计算首先应用禁用保护规则，再结合用频需求的工作频率、用途优先级、同型号通信设备的特殊干扰条件和已配置的频率判定开关，生成每条“建议”。不具备工作频率的记录基于“频段范围”和“频点数量”给出可解释的待确认建议。

空间传播、坐标、路径损耗和接收机灵敏度不在本次输入格式范围内；后续算法接入时可在独立扩展模型中加入。

## 错误处理与测试

导入时分别报告不支持的格式、缺少必填字段、空文件和无法解析的频率表达式。已成功导入的一类数据在另一类导入失败时保留，不被清空。

新增核心解析测试，覆盖 Word 表格、Excel/CSV 规则表、JSON、TXT、单位换算、复合频率和部分无效规则。更新网页和桌面端测试，覆盖两个导入控件、启动禁用条件、模拟双数据和无算法选择控件。现有 Excel 导出测试继续确保仅写入“建议”列。
