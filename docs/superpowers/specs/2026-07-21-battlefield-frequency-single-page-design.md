# 战场频谱智能指配单页重构设计

## 目标

将现有多页面 Streamlit 原型重构为单页三栏工作台，并为 PySide6 桌面 App 提供相同的信息结构和交互流程。系统用于展示战场用频需求的原始冲突、算法优化后的冲突变化及频率建议导出；当前算法为可替换的演示适配器。

## 范围

- 保留网页端与桌面 App，两端共享数据解析、演示算法接口、结果生成和导出逻辑。
- 删除物理拓扑、坐标图、旧侧边栏和多页面/底部导航。
- 深蓝渐变背景，采用工程化、可读的科技感风格。
- 不在页面展开 Excel 全表；结果通过下载按钮交付。
- 不实现真实频率冲突算法；未来通过统一接口接入。

## 输入与输出

### 输入

1. 用户导入 `.xlsx`（后续可兼容 CSV/JSON）或生成模拟数据。
2. 实际 Excel 为 `xx用频需求表`，共 12 列：序号、用频单位、型号名称、配装平台、装备数量、用途、频段范围、工作频率、频点数量、发射带宽、发射功率、建议。
3. 左侧可编辑既有七项任务参数：频道占用率、通信链路数量、用频设备数量、链路间干扰数量、最优化后剩余干扰数、频谱指配实际跨度、频谱指配跨度比例。
4. 可选择贪婪算法、DQN-GNN、遗传算法或禁忌搜索。

### 输出

1. 中间区域显示原始冲突与优化后冲突的模拟结果。
2. 新建 `*_优化结果.xlsx`，保留原始 12 列并填充“建议”列：
   - 无需调整：`保持原工作频率`。
   - 需要调整：`建议调整为 <频率范围>`，例如 `建议调整为 5.15–5.35 GHz`。
3. 右侧提供下载按钮，并展示处理记录、调整建议数量与保持原频率数量。

## 页面布局

使用已确认的 A 方案：左 24% / 中 43% / 右 33%。网页端在宽屏下使用该比例；较窄窗口允许横向滚动或由桌面端最小窗口宽度保护可读性。

### 左侧：任务控制

- 参数配置采用折叠区，默认显示常用参数。
- 数据接入提供导入实际 Excel 和生成模拟数据。
- 算法下拉框与“启动频率优化”主按钮位于底部。
- 点击启动后展示约 3 秒的阶段性进度；运行期间禁用重复启动。

### 中间：冲突态势

- 数据导入成功后立即显示原始冲突（演示目标默认 10 对）。
- 顶部显示用频需求数、原始冲突数、优化后冲突数等指标。
- 每条记录用两个需求节点和中间连线表示；红线表示仍存在冲突。
- 优化完成后切换到优化后视图，已消解组合显示绿色状态，保留冲突显示红线；默认目标为 2–4 对，初始使用 3 对。
- 记录较多时使用分页，避免画布拥挤。

### 右侧：结果输出

- 不展示完整 Excel 表格。
- 显示生成结果文件名和“建议”列已完成的状态。
- 提供“下载结果 Excel”按钮。
- 底部固定显示优化前后冲突对比：优化前 10 对、优化后 3 对，以及由此计算的冲突下降率（示例为 70%）。

## 数据和算法边界

- 导入解析器保留原始 Excel 数据和工作表样式，以便导出时只填充“建议”列。
- 冲突数据与频率建议通过 `OptimizationResult` 类似的中间结构传递给两种 UI。
- 演示算法必须可复现：相同输入和算法选择产生一致的建议与冲突计数。
- 所有算法名称均保留为可选项；在真实算法接入前，使用显式标记的演示适配器生成结果。

## 异常处理

- 文件无法解析、格式不支持、缺少表头或“建议”列时，在左侧显示可操作的错误原因，不覆盖会话中已加载数据。
- 原始文件始终只读；结果写入新的 `*_优化结果.xlsx` 文件。
- 未导入数据时禁用启动按钮，并提示先导入或生成数据。
- 运行期间禁用导入、生成和重复启动；失败后恢复可用状态并展示原因。

## 验证策略

- 单元测试：Excel 导入校验、建议列填充、结果文件导出、演示结果的可复现性、冲突统计。
- 网页端冒烟测试：导入/模拟数据、启动优化、状态切换和下载按钮可见。
- 桌面端冒烟测试：窗口初始化、三个区域存在、启动状态与结果状态切换。
- 两端一致性：相同输入下，冲突计数、建议内容和导出结果相同。

## 非目标

- 当前不实现真实 DQN-GNN、遗传算法或禁忌搜索求解。
- 当前不恢复物理拓扑和坐标渲染。
- 当前不处理 Windows XP / Windows 7 运行时兼容性迁移。
